# Week 4 - Sample Microservices Load Balancing Exercises

This module focuses on implementing **API Gateway**, **Load Balancing**, and **Resilience Patterns** using the Spring Cloud ecosystem. These concepts are fundamental for building scalable and fault-tolerant microservices.

The exercises demonstrate how an API Gateway serves as the entry point for client requests, how client-side load balancing distributes traffic across service instances, and how resilience patterns improve system reliability when downstream services become unavailable.

---

# 📖 Module Overview

This module consists of three hands-on exercises.

## ✅ Exercise 1 – Edge Services: Routing and Filtering

Implemented an API Gateway using Spring Cloud Gateway to route requests and apply global request filtering.

### Topics Covered

- API Gateway
- Edge Services
- Request Routing
- Global Filters
- Request Logging

---

## ✅ Exercise 2 – Load Balancing using Spring Cloud Gateway

Configured client-side load balancing using Spring Cloud Gateway and Spring Cloud LoadBalancer.

### Topics Covered

- Spring Cloud LoadBalancer
- Client-Side Load Balancing
- Logical Service Names (`lb://`)
- Gateway Routing
- Service Discovery Concepts

---

## ✅ Exercise 3 – Resilience Patterns in API Gateway

Implemented resilience concepts using Spring Cloud Gateway and Resilience4J Circuit Breaker with fallback handling.

### Topics Covered

- Circuit Breaker Pattern
- Resilience4J
- Fault Tolerance
- Fallback Mechanism
- API Gateway Resilience
