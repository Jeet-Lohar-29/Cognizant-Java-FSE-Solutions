# Exercise 10 - MVC Pattern

## Objective

Implement the Model-View-Controller (MVC) Design Pattern for managing student records.

## Scenario

A student management application should separate data (Model), user interface (View), and business logic (Controller).

## Files

- Student.java
- StudentView.java
- StudentController.java
- MVCPatternDemo.java

## How to Compile

```bash
javac *.java
```

## How to Run

```bash
java MVCPatternDemo
```

## Design Pattern Used

Model-View-Controller (MVC)

## Learning Outcome

- Separation of Concerns
- Model
- View
- Controller
- Loose Coupling
